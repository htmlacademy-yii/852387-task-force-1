# Классы - процессы

1) User - регистрация нового пользователя
2) User_role - смена роли
3) User_status - смена статуса, занят над задачей или свободен
4) Specialization - отмечаем какие специализации выбраны
5) Notification - уведомления: инициализация, отправка и добавление событий в ленту (наверное нужно разбить ещё на класса?)
6) Photo - добавление фото выполненных работ
7) Page_visit_count - обновление счетчика, сколько раз страница пользователя просматривалась
8) Last_activity_at - учет(обновление) времени посещения
9) Task - добавление нового задания
10) Comment - добавление нового комментария (переписка)
11) Response - добавление нового отклика на задачу
12) User_task_bind - привязка исполнителя к задаче
13) Task_status - обновление состояния, переход задания в следующий статус
14) Review - добавление отзыва со стороны заказчика при завершении задачи
15) User_rating - учет и обновление рейтинга пользователя
